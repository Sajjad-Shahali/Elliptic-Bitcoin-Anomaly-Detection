The Problem: Detecting Illicit Bitcoin

Why Bitcoin is Attractive for Crime

  • 🔗Pseudonymous — wallet addresses expose no real identity; anyone can create unlimited wallets
  • 🌐Borderless — funds move globally in minutes, bypassing banking jurisdiction
  • ♾️Irreversible — confirmed transactions cannot be clawed back by authorities
  • 💸Used for ransomware, darknet markets, mixing/tumbling services, ponzi schemes

Goal

Classify each Bitcoin transaction as illicit (money laundering, scam, ransomware) or licit (exchange, service, legitimate transfer) — using only 165 anonymous node features and the graph of transactions.

Three Core Technical Challenges

  • ⚖️
    Severe Class Imbalance
    Only 2% illicit overall, 6.5% in test. Accuracy is useless — 93.5% by always predicting licit, catching zero criminals.
  • 📉
    Temporal Concept Drift
    Illicit rate drops 11.6% → 6.5% train→test. Criminals adapt behavior over time — static models degrade.
  • ❓
    Anonymous Features
    Elliptic never published feature semantics — 165 columns are named lf_1…93, af_1…72. No domain knowledge available.

Illicit Transaction Categories in Dataset

🔒 Ransomware

Primary metric: F1 on illicit class. Accuracy meaningless at 6.5% base rate. F1 = 2·P·R/(P+R) balances catching criminals (recall) vs. not flooding analysts with false alarms (precision). PR-AUC used for threshold-independent comparison.

Methodology: Temporal Split is Critical

🚫 No random shuffle. Time-series data — shuffling leaks future patterns into training and inflates all metrics by 5–10 F1 points. Many published papers on Elliptic fall into this trap; our split is strict and methodologically correct.

Root cause of failure:
Money launderers deliberately structure transactions to look normal — fixed amounts, predictable timing, typical fan-in/fan-out ratios (smurfing, layering, mixing). Classical density-based detectors measure "how different" a point is from the majority — but illicit txs are engineered NOT to be different. They follow repeated laundering templates, so they can look more regular than licit transactions. They ARE the majority pattern in their operational context.
Spectral embedding failure:
Fiedler value ≈ 2.0 — the transaction graph is bipartite-like with no sparse graph cut. Money laundering chains are not isolated spectral communities — they are threaded through the same global topology as licit transactions. Spectral methods detect community structure, not individual node anomalies.

β-VAE: Effect of KL Weight on Performance

Lower β=0.01 wins — reconstruction-dominated. Higher β forces KL regularization which merges licit+illicit in latent space. Best unsupervised: F1=0.407.

DOMINANT — Graph AE Architecture

  • 🔮2-layer GCNConv encoder reconstructs node attributes (MSE) + graph edges (BCE)
  • 📊Joint score: α·attr_err + (1-α)·struct_err per node
  • ✅Best unsupervised — F1=0.212 · AUC=0.669 — graph topology carries signal

Trained on licit-only transactions → learns to compress/reconstruct "normal" behavior. Theory: illicit txs should give high reconstruction error because the model never learned their patterns.

Step 2 — The Surprise: Illicit transactions actually reconstruct with LOWER error than licit ones. Why? Money laundering operations use repetitive, machine-like templates — fixed amounts, fixed fan-out, systematic timing. The autoencoder finds them easy to compress. Fix: anomaly score = −MSE (inverted). Auto-detected by comparing ROC-AUC(score) vs ROC-AUC(−score) on the validation set.

Reconstruction Error Distribution (Proof)

Orange (illicit) peaks at LOWER MSE than blue (licit) — confirms inversion. The two distributions overlap significantly, explaining the F1 ceiling.

What Works

  • 🌲GBM + topology features — labels drive all performance
  • 🕸️GraphSAGE 3-layer + LayerNorm + self-loops → best GNN
  • 🔀Pseudo-labels: +3.1 F1 for GNN at p≥0.90 threshold
  • 📊Temporal split mandatory — random split inflates +5–10 F1
  • 🔍SHAP triple-overlap: lf_53, lf_90, af_70 dominate all model families

⚠️ Pitfalls Discovered

  • 🚫Threshold leakage → false F1=1.0 on val; many AML papers do this
  • 🔄AE score inversion — illicit txs reconstruct better, not worse
  • 📡GAT + sparse graph — attention collapses at avg degree 2.3
  • 📉Spectral clustering — Fiedler≈2, no anomaly community signal

Future Work

  • 🔬GNNExplainer — per-node attribution for flagged transactions
  • 📊Bootstrap F1 + McNemar tests — statistical significance across paradigms
  • 🌐DOMINANT longer training — currently undertrained (150 epochs)
  • 🤝Ensemble GBM+GNN — late fusion probability averaging
  • 🕒Temporal GNN — TGNN / EvolveGCN for concept drift adaptation
  • 📝Workshop paper — KDD/ECML-PKDD Applied DS track; 48-exp benchmark is the contribution

Core Takeaway

Supervised labels are indispensable in blockchain forensics. Illicit transactions mimic licit behavior — they are structurally normal but behaviorally criminal. No amount of unsupervised anomaly detection closes this gap. The 48-experiment benchmark quantifies the gap precisely: 0.40 vs 0.83 F1, unsupervised vs supervised.
